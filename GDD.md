# GDD — Колобок

**Жанр:** тамагочи + «скорми реальную еду с фото» + лёгкие активности (тап / резка / раннер).  
**Тон:** взрослый ироничный братан («бро», «кайф», «не катит»), без сюсюканья, без мата, **без осуждения еды** — шутка и польза, не нотация.

**Платформы:** Telegram Mini App (основная), браузер (отладка).  
**Стек:** vanilla JS (ES modules), HTML/CSS, Canvas 2D, `localStorage` + **Firebase** (облако, прокси Gemini). Сборка: `npm run build` → `js/bundle/tg.bundle.js`. Версия: `js/build.js` → `BUILD`.

**Источник правды по цифрам:** `js/config.js`, `js/phrases.js`, `js/state.js`.  
**Деплой TG:** GitHub Pages `powerviking145-lgtm.github.io/kolobok/`. **Gemini:** только через Cloud Function `geminiFoodPhoto` (EU), ключ в `GEMINI_API_KEY` — см. `FOOD_PHOTO.md`.

## 0. Видение (куда идём)

### 0.1. Главная фишка — не тапалка

**Ядро:** игрок **кормит Колобка реальной едой** (фото → ИИ → узнаёт продукт → колобок комментирует и даёт совет). Через тамагочи приучаем **фоткать еду каждый день** — задел под будущего **помощника по питанию** (калории, привычки, разбор тарелки).

| Слой | Роль |
|------|------|
| **Кормление (фото)** | Главная петля, 1–3 раза в день, рост статов, советы, прогресс |
| **Тап / резка / раннер** | Поддержка, когда лень кормить — «потупить», чуть поднять статы и очки |
| **Миниигры (битва колобков)** | Позже; **статы с кормления** = сила в бою |
| **Социалка** | Лидерборд тамагочи из Firebase |

### 0.2. Целевая дневная петля

1. Зашёл → видишь возраст колобка, статы, «голоден / сыт».
2. **Сфоткал еду** → Gemini (лимит!) → «Зашло!» + фраза колобка + буст статов + **очки прокачки** (план).
3. Между кормлениями — тап/свайп/раннер без давления.
4. Раз в день минимум — иначе статы падают; при нуле — **смерть** → «Испечь нового» (план).

### 0.3. Ограничение Gemini (деньги)

- Каждый анализ фото = платный запрос → **жёсткий дневной лимит** (план: 2–3 распознавания / сутки на игрока).
- Вне лимита: колобок **сыт / обжорел / «фотоанализ на сегодня выдохся»** — UI без сырого JSON, шутливые реплики из `phrases.js`.
- Кулдаун «сыт» уже есть: `feedCooldown` (см. §6).

### 0.4. Прогрессия

- **Статы** (сытость, жажда, здоровье, настроение) — выживание и база для будущих боёв.
- **Очки ⭐** на экране — кликер + еда на сцене + раннер (вторично).
- **Прокачка от кормления** (план): за фото-еду — отдельная валюта / XP по `kind` продукта (`good` / `neutral` / `bad`), не дублировать награду с летающими яблоками.

### 0.5. Социалка (план)

Лидерборд в Firebase: ник, **дней жизни** колобка, рекорд раннера, лучший тап-серия, **показатель кормления** (идея: «сытых дней подряд» + «разных блюд за неделю» + «лидер по health-streak» — один композитный «индекс мякиша»).

### 0.6. Смерть и возраст (план)

- Окно «Колобок съехал / испёкся» + кнопка **«Испечь нового»** (новый save, опционально в облаке).
- **Возраст** на главном: минуты → часы → дни (`bornAt` в save).

### 0.7. Туториал

- Текущий туториал (анимации, визуал) — **не ломать**.
- Добавить / переставить шаги под **фото-кормление** как главную кнопку (после этапа баланса).

### 1.2. Кормление фото (`foodPhoto` + `foodPhotoFeed.js`)

**Кнопка:** «Сфоткать еду» / кормление в футере.

| Шаг | Поведение |
|-----|-----------|
| 1 | Камера / галерея (`#food-photo-file`, `capture` на «Снять») |
| 2 | Сжатие JPEG, POST на `geminiFoodPhoto` (EU) |
| 3 | JSON: `id` из `homeFoods.list`, `comment`, `confidence` |
| 4 | Подтверждение 3 вариантов (если confidence &lt; порога) |
| 5 | «Зашло!» → буст статов по `effectsByKind`, реплика, кулдаун |

**Ошибки:** человекочитаемые (ключ, перегрузка 503, сеть) — не сырой JSON в релизе (BUILD 139+).

**Файлы:** `foodPhotoGemini.js`, `foodPhotoFeed.js`, `firebase/functions/index.js`.

### 1.3. Кулдаун «Сыт» (`feedCooldown`)

| Параметр | Значение (config) |
|----------|-------------------|
| Прод | 20 ч (`durationMs`) |
| Dev | 1 мин (`useDevDuration: true` сейчас) |
| UI | Кнопка «Сыт», тост «Покормить через {time}» |

→ Основа для **дневного лимита Gemini**: расширить счётчик «анализов за сутки», не только таймер сытости.

### 1.4. Вторичные петли (как есть)

- **Летающая еда:** тап +1 ⭐, свайп-резка +6 ⭐, bad = 0 ⭐ — §7 старого GDD.
- **Кликер:** пустой тап / колобок +1–3 ⭐, +mood.
- **Раннер:** `#btn-run`, `runScore`, рекорды дистанции.
- **Распаковка чека:** демо, `purchase.testMode: true` — не ядро продукта.

### 1.5. Статы и сохранение

- 4 стата 0–120, decay ~1% / 90 мин на доме.
- Save v7, `kolobok_save` + облако.
- Падение статов → mood/hunger визуал на колобке.

## 6. Кормление — дизайн (целевой)

### 6.1. Награды за фото-кормление

| `kind` | Статы (сейчас в config) | План: прокачка |
|--------|-------------------------|----------------|
| good | hunger +14, thirst +8, health +10, mood +6 | +XP «здоровый мякиш» |
| neutral | hunger +12, mood +10 | +XP «база» |
| bad | hunger +16, mood +14, health −4 | +XP «чит-дей» (шутка, не стыд) |

`tapScorePoints: 3` за успешное кормление — оставить умеренным.

### 6.2. Лимиты (план, согласовать)

| Лимит | Предложение |
|-------|-------------|
| Gemini / день | **2–3** успешных анализа на `telegramId` |
| Между кормлениями | Таймер «сыт» 6–8 ч **или** только дневной cap |
| UI | «Колобок обжорел, фотоанализ спит до завтра, бро» |

Хранение: Firestore `users/{id}.feedDaily` + local fallback.

### 6.3. Связь с летающей едой

- Spawn-еда — **тренировка** тапа/резки, мелкие ⭐.
- **Настоящее кормление** — только фото (или 1 бесплатный «перекус» без ИИ — опционально позже).

## 17. Критика и открытые вопросы

**Сильные стороны идеи:** понятный daily hook (фото → эмоция колобка → совет), отличие от «ещё одной тапалки», задел под nutrition assistant.

**Риски:**

1. **Зависимость от Gemini** — лимит + дружелюбный «сыт» обязательны; иначе игрок = заложник API.
2. **Два кормления** — spawn-еда vs фото путают; со временем spawn только для ⭐, не для «насыщения».
3. **Смерть** — только если мягко и редко; иначе уйдут после первого «съехал».
4. **Лидерборд** — нужна одна понятная метрика, не цифры ради цифр.

**Вопросы к тебе (ответь когда удобно):**

1. **Сколько фото-анализов в день?** 2, 3 или 1 жёстко?
2. **Смерть:** один стат в нуле или все четыре ниже 10%?
3. **Прокачка:** одна валюта «⭐ мякиша» или отдельно «сила / здоровье / наглость» от типа еды?
