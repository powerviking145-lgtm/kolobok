# GDD — Колобок

**Коротко:** Kolobok — это **тамагочи про питание**. Ты кормишь колобка через **фото своей реальной еды/напитков**, он дает **короткий нутри-совет**, а ты качаешь прогресс и тратишь ⭐ на улучшения.  
**Жанр:** тамагочи + «скорми реальную еду с фото» + лёгкие активности (тап / резка / раннер).  
**Тон:** взрослый ироничный братан («бро», «кайф», «не катит»), без сюсюканья, без мата, **без осуждения еды** — шутка и польза, не нотация.

**Платформы:** Telegram Mini App (основная), браузер (отладка).  
**Стек:** vanilla JS (ES modules), HTML/CSS, Canvas 2D, `localStorage` + **Firebase** (облако, прокси Gemini). Сборка: `npm run build` → `js/bundle/tg.bundle.js`. Версия: `js/build.js` → `BUILD`.

**Источник правды по цифрам:** `js/config.js`, `js/phrases.js`, `js/state.js`.  
**Деплой TG:** GitHub Pages `powerviking145-lgtm.github.io/kolobok/`. **Gemini:** только через Cloud Function `geminiFoodPhoto` (EU), ключ в `GEMINI_API_KEY` — см. `FOOD_PHOTO.md`.

---

## Питч

`Kolobok` — Telegram-тамагочи, который превращает питание в ежедневную игру:  
игрок фоткает свою еду, Kolobok распознаёт её через Gemini, даёт короткий полезный комментарий, а прогресс и ⭐ уходят в развитие питомца.

**Почему это работает:**

- простой daily hook: «перед тем как поесть — сфоткай»;
- эмоциональная привязка через живого персонажа (тамагочи);
- полезный фидбек по питанию без душноты;
- мета-прогресс (дейлики, прокачка, возраст, социальные цели).

**Монетизируемая ценность продукта:** высокая частота возврата (daily), длинная LTV-петля (уход, рост, смерть/ребейк, социалька), и потенциал перехода в полноценного nutrition assistant.

## 0. Видение продукта

### 0.0. Что игрок должен понять за 10 секунд

1. Это **мой живой колобок**, за ним надо следить как за тамагочи.
2. Главная кнопка — **сфоткать еду и покормить**.
3. Колобок не просто «ест», а дает **короткие советы по питанию** на основе сканов.
4. За действия я получаю ⭐ и трачу их на **прокачку/улучшения**, чтобы дольше жить и быть сильнее.

### 0.1. Главная фишка — не тапалка

**Ядро:** игрок **кормит Колобка реальной едой** (фото → ИИ → узнаёт продукт → колобок комментирует и даёт совет). Через тамагочи приучаем **фоткать еду каждый день** — задел под будущего **помощника по питанию** (калории, привычки, разбор тарелки).

| Слой | Роль |
|------|------|
| **Кормление (фото)** | Главная петля, 1–3 раза в день, рост статов, советы, прогресс |
| **Тап / резка / раннер** | Поддержка, когда лень кормить — «потупить», чуть поднять статы и очки |
| **Миниигры (битва колобков)** | Позже; **статы с кормления** = сила в бою |
| **Социалка** | Лидерборд тамагочи из Firebase |

### 0.2. Core Loop (дневная петля)

1. Зашёл → видишь возраст колобка, статы, «голоден / сыт».
2. **Сфоткал еду/напиток** → Gemini (лимит!) → «Зашло!» + факт/совет от колобка + буст нужной статы.
3. Закрыл 3 дейлика (кормить/поить/активности/очки) → забрал ⭐ награду дня.
4. Между кормлениями — тап/свайп/раннер без давления.
5. ⭐ тратятся на прокачку в магазине; растут потолки статов и выживаемость.
6. Если забил на уход — статы падают; при нуле здоровья — **смерть** → «Испечь нового».

### 0.3. Ограничение Gemini (деньги)

- Каждый анализ фото = платный запрос → **жёсткий дневной лимит** (план: 2–3 распознавания / сутки на игрока).
- Вне лимита: колобок **сыт / обжорел / «фотоанализ на сегодня выдохся»** — UI без сырого JSON, шутливые реплики из `phrases.js`.
- Кулдаун «сыт» уже есть: `feedCooldown` (см. §6).

### 0.4. Прогрессия

- **Статы** (сытость, жажда, здоровье, настроение) — выживание и база для будущих боёв.
- **Очки ⭐** на экране — суммарная валюта прогресса (тап, еда на сцене, раннер, дейлики).
- **Трата ⭐** — основной sink: прокачка статов и косметика/дома.
- **Фото-кормление** — главный источник «смысла»: улучшает состояние колобка и формирует советы по питанию.

### 0.5. Социалка (план)

Лидерборд в Firebase: ник, **дней жизни** колобка, рекорд раннера, лучший тап-серия, **показатель кормления** (идея: «сытых дней подряд» + «разных блюд за неделю» + «лидер по health-streak» — один композитный «индекс мякиша»).

### 0.6. Смерть и возраст (план)

- Окно «Колобок съехал / испёкся» + кнопка **«Испечь нового»** (новый save, опционально в облаке).
- **Возраст** на главном: минуты → часы → дни (`bornAt` в save).

### 0.7. Туториал

- Текущий туториал (анимации, визуал) — **не ломать**.
- Первый вход должен вести игрока через **обязательный сценарий фото-кормления** (без необходимости самому искать еду для первого шага).
- После флоу кормления показать, что тап/свайп и раннер — вторичные активности для очков и мини-игр.

| Шаг | Поведение |

| 4 | Подтверждение 3 вариантов (если confidence &lt; порога) |
| 5 | Экран перед кормлением: факт + короткий совет по питанию |
| 6 | Нажатие «Покормить» → буст статов, анимация роста, запись в паттерны питания |

**Ошибки:** человекочитаемые (ключ, перегрузка 503, сеть) — не сырой JSON в релизе (BUILD 139+).

**Файлы:** `foodPhotoGemini.js`, `foodPhotoFeed.js`, `firebase/functions/index.js`.

### 1.3. Лимиты и кулдаун кормления (`feedCooldown`)

| Параметр | Значение |
|----------|----------|
| Статус | Временно отключено для теста |
| Цель системы | Контроль стоимости Gemini и ритма кормления |
| UI | Понятный текст без фрустрации, когда лимит исчерпан |

→ Основа для **дневного лимита Gemini**: расширить счётчик «анализов за сутки», не только таймер сытости.

### 1.4. Вторичные петли (как есть, но не ядро)

- **Летающая еда:** тап +1 ⭐, свайп-резка +6 ⭐, bad = 0 ⭐ — это поддержка активности.
- **Кликер:** пустой тап / колобок +1–3 ⭐, +mood — «короткая сессия».
- **Раннер:** `#btn-run`, `runScore`, рекорды дистанции — отдельная активность и источник очков.
- **Распаковка чека:** демо, `purchase.testMode: true` — не ядро продукта.

### 1.5. Дейлики (реализовано v1)

3 миссии в день с прогрессом в реальном времени:

- кормить/поить через фото;
- раннер;
- тапы/свайпы;
- набор очков.

Награда за полный комплект — ⭐ с **градацией по сложности миссий** (`easy/medium/hard`).

### 1.6. Статы и сохранение

- 4 стата 0–120, decay ~1% / 90 мин на доме.
- Save v7, `kolobok_save` + облако.
- Падение статов → mood/hunger визуал на колобке.

## 11. Туториал

**Не трогать:** анимации, spotlight, видео колобка.

**Целевой сценарий первого входа (план):**

| Шаг | Что видит игрок | Что объясняем |
|-----|------------------|---------------|
| 1 | Рождение Колобка + реплика «Хозяин, я голоден. Накорми и напои меня» | Это тамагочи: за ним нужно регулярно ухаживать |
| 2 | Подсветка кнопки фото-кормления | Главный loop: сфоткал еду -> получил совет -> нажал `Покормить` |
| 3 | Экран выбора фото (камера/галерея) | На Android может открываться только галерея — это допустимое поведение |
| 4 | Fallback “я нашел еду/воду, но только на первый раз” | Игрок не должен застрять в первом запуске |
| 5 | Экран сомнения распознавания (`Колобок не понял`) | Как выбрать правильный вариант вручную из списка |
| 6 | Экран факта/совета + CTA `Покормить` | Колобок дает короткие рекомендации по питанию |
| 7 | Анимация роста сытости/жажды | Результат действия понятен мгновенно |
| 8 | Финальная фраза про регулярность | Чем чаще кормишь, тем ценнее советы и точнее оценка рациона |
| 9 | Короткий блок про тап/свайп/раннер | Это “снять напряжение”, фарм очков и вход в мини-игры `Сжечь калории` |

**Важно для UX:**

- На первом входе не заставлять игрока искать реальную еду самому.
- Любой “непонятный” этап объяснять коротко, шутливо и без технической лексики.
- Core loop должен считываться раньше, чем игрок увидит вторичные механики.

## 17. Критика и открытые вопросы

**Сильные стороны идеи:** понятный daily hook (фото → эмоция колобка → совет), отличие от «ещё одной тапалки», задел под nutrition assistant.

**Риски:**

1. **Зависимость от Gemini** — лимит + дружелюбный «сыт» обязательны; иначе игрок = заложник API.
2. **Конкуренция loop-ов** — игрок не должен путать «поиграл в тап/свайп» с «покормил по-настоящему через фото».
3. **Смерть** — только если мягко и редко; иначе уйдут после первого «съехал».
4. **Лидерборд** — нужна одна понятная метрика, не цифры ради цифр.

**Вопросы к тебе (ответь когда удобно):**

1. **Сколько фото-анализов в день?** 2, 3 или динамический лимит по retention?
2. **Смерть:** точный триггер и частота, чтобы не ломать мотивацию.
3. **Экономика ⭐:** доля награды из дейликов vs раннера vs тапов.
4. **Прокачка:** только ⭐ или добавить вторичный nutrition-XP позже.
